# Session Handoff — 2026-03-14

## What Was Done

### LFO System Extensions
- **Slow sync divisions**: `LFO_SYNC_8_1` / `16_1` / `32_1` (8/16/32 bars) in `synth.h` enum + `getLfoRateFromSync()`. DAW cycles through 11 sync names.
- **Per-LFO phase offset**: 4 fields on `SynthPatch` (`p_filterLfoPhaseOffset` etc), applied as initial phase in `resetVoiceLfos()`. 0.5 = inverted sine.
- **FM mod index LFO**: 5th LFO — `fmLfoRate/Depth/Shape/Sync/PhaseOffset` on patch, `fmLfo*` fields on Voice. Applied in `processVoice()` before oscillator switch, restored after. Only shown in DAW when `waveType == WAVE_FM`.
- **Compact LFO UI**: `ui_col_float_pair()` and `ui_col_cycle_float_pair()` in `daw_widgets.h`. Rate+Depth on one row, Sync+Phase on one row. Columns widened (col5: 575→620, col6: 720→760).

### FM Scaling Fix
- `processFMOscillator()` in `synth_oscillators.h` now divides modIndex by 2π before applying to phase. ModIndex is now standard β (radians), matching FM synthesis literature.
- All existing presets multiplied by 6.283 to preserve sound. Same for bridge songs (`songs.h`) and all `.song` files. Default `p_fmModIndex` changed from 1.0 to 2.0. DAW slider range 0-30.
